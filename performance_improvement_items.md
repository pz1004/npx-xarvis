# 성능 개선 항목 도출 보고서

## 1. 개요
코드 감사 결과와 문헌 조사 결과를 바탕으로, 제안된 알고리즘과 구현 코드의 성능을 개선하고 고품질 저널 투고 가능 수준으로 끌어올리기 위한 구체적인 개선 항목을 도출합니다.

## 2. 소프트웨어 구현 및 실행 성능 개선

### 2.1 Numba JIT 컴파일 적용 (필수)
- **문제점:** 현재 `proposed_balanced.py`의 `apply()` 메서드는 순수 Python `for` 루프로 구현되어 있어, 수백만 개의 이벤트를 처리할 때 심각한 병목 현상이 발생합니다.
- **개선 방안:** 핵심 필터링 로직을 Numba의 `@njit` 데코레이터를 사용하여 컴파일합니다. 상태 배열(`t_raw_last`, `p_raw_last` 등)과 이벤트 배열을 Numba 호환 형태로 전달하여 처리 속도를 C/C++ 수준으로 향상시킵니다.
- **기대 효과:** 전처리 지연 시간(preprocessing latency) 대폭 감소, 대규모 데이터셋(CIFAR10-DVS 등)에 대한 실험 실행 시간 단축.

### 2.2 실험 설정 일치화
- **문제점:** `implementation.md`는 5개의 시드(`3407, 3413, 3421, 3433, 3449`)를 요구하지만, `configs/training/default.json`은 3개의 시드(`[0, 1, 2]`)로 설정되어 있습니다. 논문 `main.tex`는 3개 시드를 명시하고 있습니다.
- **개선 방안:** 논문 `main.tex`의 명시에 따라 3개 시드를 유지하되, `implementation.md`의 요구사항과 충돌하지 않도록 명확히 정리하거나, 통계적 유의성을 높이기 위해 논문과 설정을 모두 5개 시드로 수정합니다. (저널 투고를 위해서는 5개 시드가 더 권장됨)

## 3. 알고리즘 및 모델 성능 개선

### 3.1 적응형 임계값 (Adaptive Thresholding) 도입 고려
- **문제점:** 현재 필터는 고정된 $K_0$ 및 $\gamma$ 파라미터를 사용합니다. 이는 조도 변화나 동적 환경에서 최적의 성능을 내기 어렵습니다.
- **개선 방안:** 지역 이벤트 밀도(local event density)나 EWMA 기반의 `rate_hz`를 활용하여 임계값을 동적으로 조절하는 메커니즘을 추가합니다.
- **기대 효과:** 다양한 노이즈 환경에서의 강건성(robustness) 향상.

### 3.2 SNN 백본 아키텍처 강화
- **문제점:** 현재 제공된 `EventSNN`은 기본적인 Conv-LIF 구조입니다. 최신 연구(SpiLiFormer, STCA-SNN 등)는 어텐션 메커니즘이나 시간적 정규화를 통해 훨씬 높은 정확도를 달성하고 있습니다.
- **개선 방안:** 
  - 시간적 정규화(Temporal Regularization) 기법 적용
  - 측면 억제(Lateral Inhibition) 메커니즘 추가
  - (선택적) 가벼운 어텐션 모듈 통합
- **기대 효과:** CIFAR10-DVS 및 N-Caltech101과 같은 복잡한 데이터셋에서의 분류 정확도 향상.

### 3.3 차분 코딩 (Differential Coding) 적용
- **문제점:** SNN의 스파이크 통신량이 많아지면 에너지 효율이 떨어집니다.
- **개선 방안:** 신뢰도 코딩(Confidence Coding)과 결합하여, 상태 변화가 있을 때만 스파이크를 전송하는 차분 코딩 방식을 SNN 입력단에 적용합니다.
- **기대 효과:** 시냅스 연산(SOPs) 수 감소 및 에너지 효율 향상.

## 4. 평가 메트릭 및 벤치마크 개선

### 4.1 구조적 보존 메트릭 추가
- **문제점:** 현재 평가는 주로 분류 정확도와 노이즈 제거율(ROC/AUC)에 의존합니다.
- **개선 방안:** 최신 문헌에서 제안된 ESR (Event Structural Ratio) 또는 AOCC 메트릭을 평가 파이프라인에 추가하여, 필터링 후 객체의 구조적 정보가 얼마나 잘 보존되는지 정량적으로 측정합니다.
- **기대 효과:** 필터의 우수성을 다각도로 입증하여 논문의 설득력 강화.

### 4.2 실제 노이즈 데이터셋 검증
- **문제점:** 합성 노이즈(Synthetic noise)에 대한 평가는 통제된 환경에서의 성능만 보여줍니다.
- **개선 방안:** LED (Large-scale Event Denoising) 데이터셋과 같은 실제 노이즈가 포함된 데이터셋에 대한 정성적/정량적 평가 결과를 추가합니다.
- **기대 효과:** 실제 환경에서의 적용 가능성 입증.

## 5. 결론 및 우선순위
가장 시급하고 실현 가능한 개선 항목은 **Numba JIT 컴파일 적용**과 **실험 설정 일치화**입니다. 이를 통해 실험을 빠르게 수행할 수 있는 기반을 마련한 후, **SNN 백본 강화**와 **구조적 보존 메트릭 추가**를 통해 논문의 질적 수준을 높이는 방향으로 진행해야 합니다.
